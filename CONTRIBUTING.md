# Contributing to Fault×32

Thanks for your interest in contributing! All contributions must follow these rules:

## 1. Contributor License Agreement (CLA)
- Before submitting a PR, you must read and agree to the [CLA](CLA.md) and FCSL v1.1 license.
- By contributing, you agree your work can be used in Fault×32 under these rules, and credit will be preserved.

## 2. How to Submit a Pull Request
1. Fork the repo or work in a private branch (with explicit permission if needed).  
2. Make your changes.  
3. Refractor your code if necessary to keep style clean.  
4. Submit a pull request to the `dev` branch (or a branch specified by maintainers).  
5. Wait for approval — all PRs are **reviewed before merge**.  

## 3. Rules for Code & Assets
- Do **not** copy or fork the repo publicly — you need explicit permission.  
- Tutorials or snippets are allowed for educational purposes.  
- Your contribution becomes part of Fault×32 under FCSL v1.1 rules once merged.  
- You retain ownership of your original code, but it must not include Project-owned libraries outside of Fault×32.  

## 4. Branches & Commits
- Use `dev` for development and `main` for releases.  
- **Force pushes are blocked** on main branches.  
- **Web commits require sign-off** (add `Signed-off-by:` line in the commit message) to acknowledge the CLA.  

## 5. Attribution & Credit
- All contributors must be credited in `README.md` and `CREDITS.md`.  
- Do not remove existing copyright, license, or attribution.  

## 6. Reporting Issues & Ideas
- Use **GitHub Issues** for bugs, feature requests, or suggestions.  
- Use **Discussions** for community chat or educational questions.  

Thank you for helping make Fault×32 awesome!
